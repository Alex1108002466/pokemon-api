# Pokemon API

Веб-API на FastAPI для получения и управления информацией о покемонах.
Данные хранятся в SQLite, миграции структуры БД — через Alembic.

## Стек технологий

- [FastAPI](https://fastapi.tiangolo.com/) — веб-фреймворк
- [SQLModel](https://sqlmodel.tiangolo.com/) — ORM (работа с базой данных)
- [Alembic](https://alembic.sqlalchemy.org/) — миграции базы данных
- [uv](https://docs.astral.sh/uv/) — менеджер зависимостей
- SQLite — база данных
- Docker — контейнеризация

## Эндпоинты

|  Метод |      URL        |         Описание            |
|--------|-----------------|-----------------------------|
| GET    | `/`             | Проверка работы сервера     |
| GET    | `/pokemon`      | Список всех покемонов       |
| GET    | `/pokemon/{id}` | Информация о покемоне по id |
| POST   | `/pokemon/`     | Создать нового покемона     |
| PATCH  | `/pokemon/{id}` | Обновить покемона           |
| DELETE | `/pokemon/{id}` | Удалить покемона            |

Полная интерактивная документация (Swagger) доступна по адресу `/docs` после запуска сервера.

## Способ 1 — запуск напрямую (без Docker)

Подходит для локальной разработки.

```bash
# Установить зависимости
uv sync

# Применить миграции (создаст pokemon.db со всеми таблицами)
alembic upgrade head

# Запустить сервер
uvicorn app.main:app --reload
```

При первом запуске сервер автоматически заполнит базу тремя покемонами (pikachu, charizard, mewtwo).

Сервер будет доступен на `http://127.0.0.1:8000`.

----------------------------------------------------------------------------------------------------

## Способ 2 — запуск через Docker

Подходит для изолированного запуска без необходимости устанавливать Python и зависимости на компьютер.

### Требования
Установленный [Docker Desktop](https://www.docker.com/products/docker-desktop/).

### Собрать образ
```bash
docker build -t pokemon-api .
```

### Запустить контейнер
```bash
docker run -d -p 8000:8000 --name pokemon-api pokemon-api
```

Миграции применяются автоматически при старте контейнера, перед запуском сервера.

### Проверить логи
```bash
docker logs pokemon-api
```

### Остановить и удалить контейнер
```bash
docker stop pokemon-api
docker rm pokemon-api
```

### Пересобрать после изменений в коде
```bash
docker stop pokemon-api
docker rm pokemon-api
docker build -t pokemon-api .
docker run -d -p 8000:8000 --name pokemon-api pokemon-api
```

Сервер будет доступен на `http://127.0.0.1:8000`, так же как и при прямом запуске.

## Работа с миграциями

Если изменилась модель `Pokemon` в `app/main.py` (добавлено/удалено поле), нужно сгенерировать новую миграцию:

```bash
alembic revision --autogenerate -m "описание изменения"
alembic upgrade head
```

Полезные команды:

```bash
alembic current       # текущая версия БД
alembic history       # история всех миграций
alembic downgrade -1  # откатить последнюю миграцию
```
